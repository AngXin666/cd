# 车队管家小程序管理后台需求文档

## 1. 系统概述

### 1.1 系统名称\n车队管家小程序管理后台

### 1.2 系统描述
专为车队管理打造的电脑端管理后台系统，与小程序前端数据打通同步，提供完整的车队运营管理和数据统计功能，支持便捷的网页访问和管理操作。

### 1.3 项目目标
开发一个功能完整、独立运行的电脑端车队管理后台系统，并与小程序前端数据进行打通和同步，实现车队运营管理的全流程数字化管理。

## 2. 技术架构要求

### 2.1 后台服务架构
- **前后端分离架构**：确立清晰的前后端分离架构，定义标准的API交互规范（RESTful API）
- **后端服务设计**：设计稳定、可扩展的后端服务，确保数据处理的高效性和安全性
- **数据库规划**：规划数据库结构，支持车队管理相关的核心业务数据存储与调用

### 2.2 API交互规范
- **接口设计**：采用RESTful API设计规范
- **数据格式**：JSON格式数据交换
- **认证机制**：基于Token的用户认证体系
- **错误处理**：统一的错误码和错误信息返回

### 2.3 环境配置
- **开发环境**：本地开发环境配置
- **测试环境**：集成测试和验收测试环境
- **生产环境**：正式上线运行环境
- **配置管理**：不同环境的数据库连接、API密钥、文件存储路径等配置参数管理

## 3. 核心功能模块

### 3.1 仪表盘功能
- **关键数据统计**：
  - 今天出勤人数统计
  - 今日总件数统计
  - 本月完成件数统计
  - 车辆使用率统计
  - 司机工作效率统计
- **数据展示形式**：采用直观的图表和卡片式布局展示关键业务指标

### 3.2 车辆管理功能
- **车辆信息管理**：
  - 车辆基本信息录入（车牌号、车型、品牌、购置日期等）
  - 车辆状态监控（已启动、已停用、维修中、正常使用等）
  - 车辆保养记录管理
  - 行驶里程统计
  - 燃油消耗记录
- **车辆分配管理**：
  - 司机与车辆匹配管理
  - 车辆使用记录跟踪
  - 车辆调度安排

### 3.3 司机管理功能
- **司机档案管理**：
  - 司机个人信息录入（姓名、身份证号、驾驶证信息等）
  - 联系方式管理
  - 从业资格证书管理
  - 司龄和工作经验记录
- **司机状态管理**：
  - 出勤状态跟踪
  - 工作排班安排
  - 休假和请假管理\n- **绩效查看**：
  - 司机工作效率统计
  - 完成任务数量
  - 服务质量评价
  - 绩效考核数据展示

### 3.4 用户管理功能
- **用户信息管理**：
  - 添加、编辑、删除用户资料
  - 角色管理（管理员、司机、仓库管理员等）
  - 状态管理（启用、禁用）
  - 权限分配
  - 通讯录管理
- **仓库管理**：\n  - 仓库信息录入和管理
  - 仓库人员分配
  - 仓库库存数据关联

### 3.5 系统设置功能
- **角色权限管理**：
  - 角色创建和权限配置
  - 用户角色分配
  - 功能权限控制
- **基础参数配置**：\n  - 系统参数设置
  - 业务规则配置
  - 默认值设定
- **操作日志审计**：
  - 用户操作记录追踪\n  - 数据变更日志
  - 操作时间戳记录

### 3.6 数据统计功能
- **用户统计**：按时间段统计用户注册和活跃情况
- **操作记录**：展示用户操作日志
- **费用统计**：统计相关费用数据
- **效率分析**：分析用户行为和系统使用效率

## 4. 系统逻辑流程

### 4.1 系统登录流程
1. 用户访问登录页面
2. 输入账号和密码
3. 系统验证账号有效性
4. 验证密码正确性
5. 检查用户角色（管理员、司机等）
6. 生成访问token
7. 跳转到对应管理首页

### 4.2 仪表盘访问流程
1. 用户登录成功后自动跳转到对应仪表盘
2. 系统根据用户角色显示个性化数据概览
3. 支持实时数据刷新和动态更新
4. 提供数据详情查看入口

### 4.3 车辆管理流程
1. 系统自动同步车辆数据
2. 支持车辆信息批量导入
3. 提供完整的车辆档案管理
4. 实时更新车辆状态和使用记录

### 4.4 司机管理流程
1. 系统自动同步司机数据
2. 支持司机信息批量导入
3. 提供完整的司机档案管理
4. 实时更新司机状态和绩效数据

### 4.5 数据统计流程
1. 系统自动收集各项业务数据
2. 按预设维度进行数据统计
3. 生成可视化的图表展示
4. 支持数据导出和打印

## 6. 安全配置要求

### 6.1 用户认证
- **登录机制**：基于用户名密码的用户认证
- **密码安全**：密码强度要求和加密存储
- **登录失败处理**：登录失败次数限制和锁定策略

### 6.2 权限校验
- **角色权限**：基于角色的权限控制
- **功能授权**：细粒度的功能访问控制
- **数据权限**：根据不同角色设置数据可见范围

### 6.3 数据加密
- **数据传输**：HTTPS加密传输
- **数据存储**：敏感数据加密存储
- **会话管理**：安全的Session和Token管理

### 6.4 API防护
- **防攻击措施**：防止SQL注入、XSS攻击等常见安全威胁
- **速率限制**：API请求频率控制
- **IP黑白名单**：限制非法IP访问

## 7. 非功能性要求

### 7.1 用户体验
- **界面设计**：操作逻辑清晰，交互顺畅，符合管理人员的工作习惯
- **响应速度**：系统响应迅速，页面加载时间优化
- **导航结构**：合理的功能分类和导航设计

### 7.2 性能与安全
- **系统性能**：保证高并发下的系统响应速度
- **数据安全**：完善的数据备份和容灾方案
- **监控预警**：系统异常监控和告警机制
\n### 7.3 技术选型
- **前端技术**：Vue.js框架，Vant UI组件库
- **后端技术**：Node.js + Express框架
- **数据库**：MySQL关系型数据库

## 9. 设计风格

### 9.1 配色方案
- **主色调**：蓝色系渐变（#1890ff到#4a90e2），体现专业可靠
- **强调色**：橙色（#ff7a45）用于重要操作按钮
- **成功提示**：绿色（#52c41a），错误提示：红色（#ff4d4f）
- **背景色**：浅灰色（#f5f5f5）作为主要背景色

### 9.2 视觉细节\n- **圆角设计**：统一使用4px圆角，营造现代商务感
- **卡片阴影**：轻微阴影效果增强层次感
- **图标风格**：线性图标配合填充图标，保持视觉一致性
- **按钮样式**：主要按钮采用填充样式，次要按钮采用边框样式

### 9.3 整体布局
- **布局方式**：采用卡片式布局，信息层次清晰
- **响应式设计**：适配不同电脑屏幕分辨率
- **导航结构**：左侧导航布局，便于功能分类管理
- **数据展示**：清晰的表格和图表形式，支持数据导出
