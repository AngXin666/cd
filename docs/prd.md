## 1. 小程序概述
### 1.1 小程序名称
车队管家

### 1.2 小程序描述
一款专为车队管理打造的微信小程序，提供多角色权限管理和多层级数据仪表盘数据库，包含超级管理员、超级管理员和驾驶员姓名三个不同界面，满足车队运营的分层管理需求。本次进行注册、登录功能的数据库性重构，采用现代化技术架构，提升数据库的可维护性、安全性和管理员体验。新增手机号直接登录功能，支持管理员使用联系电话等11位手机号直接登录数据库，登录时自动添加邮箱功能。所有页面均支持下拉手动刷新功能，提升管理员交互体验。新增智能化考勤打卡提醒与请假管理数据库，实现司机每日首次登录自动检测打卡状态、启动计件前二次检测、请假状态豁免逻辑以及假期主动撤销功能，全面提升考勤管理的智能化水平。新增管理司机作台仪表盘考勤管理功能，支持从工作台直接进入考勤管理界面查看司机总数和待审批信息，并优化交互体验和数据展示方式。优化数据库管理员对超级管理员的权限配置逻辑，实现细粒度权限控制和功能模块内同等操作权限。重构错误管理端功能模块，将品类管理升级为仓库管理，整合品类管理和考勤规则设置两大功能区。优化超级管理员权限，增加对司机基本资料的修改权限，提升管理效率和操作便利性。新增多端消息同步与提醒数据库，实现跨角色的智能消息推送、滚动提醒和信息中心管理，确保重要信息的及时传达和有效管理。优化管理端请假审批管理功能，提供更便捷的审批操作和状态跟踪。新增司机打卡记录查看界面，支持详细的考勤数据查看和统计分析。优化超级管理员后台管理员管理模块中的编辑管理员信息页面，调整字段配置和角色选择，提升管理员信息管理的针对性和实用性。新增登录邮箱自动添加功能和管理员创建成功提示优化，提升管理员体验和操作便利性。新增司机出勤统计仪表盘功能，提供全面的司机出勤率统计、个人出勤数据分析、满勤标记识别和可视化图表展示，实现数据驱动的考勤管理决策支持。新增管理员管理模块仓库切换与权限管理架构，采用左右滑动切换仓库方式，支持对应仓库在职管理员管理，并提供管理员界面显示管理员信息、管理仓库和权限范围。新增错误管理端管理员详情页面权限控制功能增强，实现基于权限的司机信息修改功能，包括权限判断、编辑按钮显示控制和完整的修改功能实现，提供与超级管理员同等的司机信息编辑能力。新增全界面滑动返回功能，支持所有界面通过右滑手势返回上一层，提供统一的导航体验和便捷的操作方式。**新增品类管理仓库切换与价格设置功能，支持双管理员端为不同仓库的纯司机和带车司机设置对应价格，实现精细化的仓库级别价格管理。**

## 2. 品类管理仓库切换与价格设置功能（新增）
### 2.1 功能概述
- **功能gps**
  - 为品类管理模块增加仓库切换能力，支持管理员为不同仓库设置独立的价格体系
  - 实现纯司机和带车司机的差异化价格设置，满足不同仓库的业务需求
  - 提供双管理员端（超级管理员和超级管理员）的统一价格管理体验
  - 支持仓库级别的价格策略制定和执行
  - 实现价格数据的仓库隔离和权限控制

- **适用范围**
  - 超级管理员端品类管理模块
  - 数据库管理员端品类管理模块
  - 所有已配置的仓库和品类
  - 纯司机和带车司机两种司机类型
  - 价格设置、查看、修改等全流程操作

### 2.2 仓库切换机制
- **仓库切换界面设计**
  - **切换方式**：采用顶部标签页形式，支持左右滑动切换仓库
  - **仓库指示器**：显示当前选中仓库名称，提供仓库数量指示（如"仓库A (1/3)"）
  - **快速切换**：支持点击仓库名称弹出仓库选择器，实现快速跳转
  - **权限验证**：根据管理员权限范围，仅显示有权限管理的仓库
  - **状态保持**：切换仓库时保持当前的品类选择和编辑状态

- **仓库数据隔离**
  - **独立价格体系**：每个仓库维护独立的品类价格设置
  - **数据同步机制**：仓库间价格数据完全隔离，避免相互影响
  - **缓存管理**：为每个仓库维护独立的价格数据缓存
  - **权限控制**：确保管理员只能访问和修改有权限的仓库价格

### 2.3 价格设置体系
- **司机类型区分**
  - **纯司机价格**：
    + 基础计件价格：设置纯司机的基础计件单价
    + 加班价格：设置纯司机的加班计件单价
    + 特殊时段价格：设置节假日、夜班等特殊时段的价格
    + 绩效奖励价格：设置基于绩效的额外奖励单价

  - **带车司机价格**：
    + 基础计件价格：设置带车司机的基础计件单价（通常高于纯司机）
    + 车辆补贴：设置车辆使用、维护等相关补贴
    + 油费补贴：设置油费补贴标准和计算方式
    + 车辆磨损补偿：设置车辆磨损的补偿价格
    + 特殊任务价格：设置需要车辆的特殊任务价格

### 2.4 价格管理界面设计
- **双管理员端统一界面**
  - **超级管理员端**：
    + 完整的价格设置权限，可设置所有仓库的价格
    + 价格审核功能，可审核和批准超级管理员的价格修改
    + 价格历史查看，可查看所有价格变更历史
    + 批量价格设置，可批量设置多个仓库的价格

  - **超级管理员端**：
    + 根据权限范围设置对应仓库的价格
    + 价格修改申请，重要价格变更需要超级管理员审核
    + 价格对比功能，可对比不同仓库的价格设置
    + 价格建议功能，基于历史数据提供价格建议

- **价格设置表单设计**
  - **分类设置区域**：
    + 司机类型选择：纯司机/带车司机切换标签
    + 价格类型分组：基础价格、补贴价格、特殊价格等分组展示
    + 批量设置功能：支持批量设置相同类型的价格
    + 模板应用功能：支持应用其他仓库的价格模板

  - **价格输入控件**：
    + 数值输入框：支持小数点后两位的价格输入
    + 单位显示：明确显示价格单位（元/件、元/小时等）
    + 实时计算：输入价格时实时计算预估成本
    + 验证提示：价格范围验证和异常提示

### 2.5 价格数据管理
- **数据存储结构**
  - **仓库-品类-司机类型三级结构**：
    + 仓库ID -> 品类ID -> 司机类型 -> 价格配置
    + 支持多维度的价格查询和统计
    + 保证数据的完整性和一致性
    + 支持价格数据的版本管理

  - **价格配置字段**：
    + 基础价格：base_price（基础计件单价）
    + 加班价格：overtime_price（加班计件单价）
    + 特殊时段价格：special_time_price（特殊时段价格）
    + 车辆补贴：vehicle_allowance（仅带车司机）
    + 油费补贴：fuel_allowance（仅带车司机）
    + 磨损补偿：wear_compensation（仅带车司机）
    + 生效时间：effective_time（价格生效时间）
    + 创建人：creator_id（价格设置人）
    + 审核状态：audit_status（审核状态）

- **价格历史管理**
  - **变更记录**：
    + 记录每次价格变更的详细信息
    + 包括变更前后价格、变更人、变更时间、变更原因
    + 支持价格变更的回滚操作
    + 提供价格变更趋势分析

  - **审核流程**：
    + 重要价格变更需要上级审核
    + 审核流程可配置，支持多级审核
    + 审核意见记录和通知机制
    + 审核超时自动提醒

### 2.6 价格应用与计算
- **计件价格应用**
  - **实时价格获取**：
    + 根据司机所属仓库和司机类型自动获取对应价格
    + 支持价格的实时更新和生效
    + 处理价格变更期间的过渡逻辑
    + 确保价格应用的准确性和及时性

  - **价格计算逻辑**：
    + 基础计件：数量 × 基础价格
    + 加班计件：加班数量 × 加班价格
    + 带车补贴：根据车辆使用情况计算补贴
    + 特殊任务：根据任务类型应用对应价格
    + 综合结算：汇总各项价格计算总收入

### 2.7 权限控制与安全

## 3. 全界面滑动返回功能
### 3.1 功能概述
- **功能gps**
  - 为小程序所有界面提供统一的滑动返回导航体验
  - 通过右滑手势实现快速返回上一层界面的操作
  - 提供与微信小程序原生体验一致的导航交互
  - 增强管理员操作的便捷性和流畅性
  - 减少对返回按钮的依赖，提升单手操作体验

- **适用范围**
  - 所有二级及以上层级的界面页面
  - 包括但不限于：管理员管理、考勤管理、请假管理、仓库管理、司机资料管理、消息中心、打卡记录、出勤统计、权限配置、品类管理等所有功能模块
  - 弹窗和模态框界面
  - 详情页面和编辑页面
  - 列表页面和统计页面

### 3.2 滑动返回交互设计
- **手势识别机制**
  - **滑动方向**：从屏幕左边缘向右滑动
  - **触发区域**：屏幕左侧边缘20px宽度区域
  - **滑动距离**：滑动距离超过屏幕宽度的年龄%时触发返回
  - **滑动速度**：快速滑动时降低触发距离要求至屏幕宽度的20%
  - **手势冲突处理**：与页面内容的横向滑动手势进行优先级区分

- **视觉反馈设计**
  - **滑动指示**：滑动过程中显示半透明的上一层页面预览
  - **进度指示**：通过页面位移程度显示返回操作的进度
  - **阻尼效果**：滑动到一定程度后增加阻尼感，提供物理反馈
  - **回弹动画**：滑动距离不足时的回弹动画效果
  - **完成动画**：成功触发返回时的页面切换动画
