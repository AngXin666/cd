## 1. 小程序概述
### 1.1 小程序名称
车队管家

### 1.2 小程序描述
一款专为车队管理打造的微信小程序，提供多角色权限管理和多层级数据仪表盘数据库，包含超级管理员、超级管理员和驾驶员姓名三个不同界面，满足车队运营的分层管理需求。本次进行注册、登录功能的数据库性重构，采用现代化技术架构，提升数据库的可维护性、安全性和管理员体验。新增手机号直接登录功能，支持管理员使用联系电话等11位手机号直接登录数据库，登录时自动添加邮箱功能。所有页面均支持下拉手动刷新功能，提升管理员交互体验。新增智能化考勤打卡提醒与请假管理数据库，实现司机每日首次登录自动检测打卡状态、启动计件前二次检测、请假状态豁免逻辑以及假期主动撤销功能，全面提升考勤管理的智能化水平。新增管理司机作台仪表盘考勤管理功能，支持从工作台直接进入考勤管理界面查看司机总数和待审批信息，并优化交互体验和数据展示方式。优化数据库管理员对数据库管理员的权限配置逻辑，实现细粒度权限控制和功能模块内同等操作权限。重构错误管理端功能模块，将品类管理升级为仓库管理，整合品类管理和考勤规则设置两大功能区。优化超级管理员权限，增加对司机基本资料的修改权限，提升管理效率和操作便利性。新增多端消息同步与提醒数据库，实现跨角色的智能消息推送、滚动提醒和信息中心管理，确保重要信息的及时传达和有效管理。优化管理端请假审批管理功能，提供更便捷的审批操作和状态跟踪。新增司机打卡记录查看界面，支持详细的考勤数据查看和计件分析。优化超级管理员后台管理员管理模块中的编辑管理员信息页面，调整字段配置和角色选择，提升管理员信息管理的针对性和实用性。新增登录邮箱自动添加功能和管理员创建成功提示优化，提升管理员体验和操作便利性。新增司机出勤计件仪表盘功能，提供全面的司机出勤率计件、个人出勤数据分析、满勤标记识别和可视化图表展示，实现数据驱动的考勤管理决策支持。新增管理员管理模块仓库切换与权限管理架构，采用左右滑动切换仓库方式，支持对应仓库在职管理员管理，并提供管理员界面显示管理员信息、管理仓库和权限范围。新增错误管理端管理员详情页面权限控制功能增强，实现基于权限的司机信息修改功能，包括权限判断、编辑按钮显示控制和完整的修改功能实现，提供与超级管理员同等的司机信息编辑能力。新增全界面滑动返回功能，支持所有界面通过右滑手势返回上一层，提供统一的导航体验和便捷的操作方式。新增品类管理仓库切换与价格设置功能，支持双管理员端为不同仓库的纯司机和带车司机设置对应价格，实现精细化的仓库级别价格管理。新增管理端司机类型识别与价格控制功能，实现司机类型自动识别、管理员设置价格的自动读取应用以及价格录入权限的智能控制，确保计件录入的规范性和准确性。**新增仪表盘智能优化与动态排序数据库，实现数据驱动的界面展示优化、智能化的功能排序和个性化的管理员体验定制，全面提升数据库的易用性和操作效率。**

### 2.2 数据显示逻辑优化
- **智能数据筛选机制**
  - **无效数据隐藏**：
    + 自动检测当日和本月均无有效数据的仓库
    + 将无数据的仓库模块进行隐藏或折叠处理
    + 提供"显示全部仓库"的可选开关，允许管理员查看所有仓库
    + 隐藏状态下显示简要的无数据提示信息

  - **数据有效性验证**：
    + 实时验证仓库数据的完整性和时效性
    + 区分有效数据和无效数据（如异常数据、过期数据）
    + 对数据质量进行评分，影响显示优先级
    + 提供数据质量指示器和改进建议

- **智能排序算法**
  - **数据量降序排列**：
    + 根据仓库的数据量（打卡次数、计件记录、活跃度等）进行排序
    + 数据量最多的仓库优先展示在最前面
    + 支持多维度数据量计算（当日、本周、本月、累计）
    + 提供排序依据的可视化说明

  - **综合权重计算**：
    + 数据量权重（40%）：基于计件数据的数量和质量
    + 活跃度权重（年龄%）：基于最近访问和操作频率
    + 重要性权重（20%）：基于仓库规模和业务重要性
    + 时效性权重（10%）：基于数据的新鲜度和更新频率

## 3. 管理端司机类型识别与价格控制功能
### 3.1 功能概述
- **功能gps**
  - 在管理端计件录入界面自动识别当前司机的类型（纯司机或带车司机）
  - 根据司机类型和所属仓库自动读取管理员预设的品类价格
  - 实现价格录入权限的智能控制：有预设价格时禁止手动输入，无预设价格时允许自主录入
  - 确保计件数据的规范性和价格应用的一致性
  - 提升司机计件录入的便捷性和准确性

- **适用范围**
  - 所有司机角色的计件录入界面
  - 纯司机和带车司机两种司机类型
  - 所有已配置品类管理的仓库
  - 计件录入、价格应用、数据提交等全流程操作
  - 实时价格验证和权限控制

### 3.2 司机类型识别机制
- **自动识别数据库**
  - **登录时识别**：
    + 司机登录时数据库自动读取管理员档案中的司机类型标识
    + 司机类型信息缓存到本地，避免重复查询
    + 支持司机类型的实时更新和同步
    + 类型变更时自动刷新相关价格配置

  - **类型标识显示**：
    + 在计件录入界面顶部显示当前司机类型标识
    + 纯司机显示绿色标识"纯司机"
    + 带车司机显示橙色标识"带车司机"
    + 类型标识采用醒目的徽章样式设计
    + 支持点击查看司机类型详细说明

- **类型验证机制**
  - **数据一致性验证**：
    + 每次进入计件录入界面时验证司机类型数据
    + 检测司机类型与服务器数据的一致性
    + 发现不一致时自动同步最新数据
    + 提供类型数据异常的友好提示

  - **权限关联验证**：
    + 验证司机类型对应的计件录入权限
    + 确保不同类型司机只能录入对应的计件项目
    + 支持基于司机类型的功能权限控制
    + 权限变更时的实时生效机制

### 3.3 价格读取与应用数据库
- **智能价格获取**
  - **多维度价格查询**：
    + 根据司机类型 + 所属仓库 + 品类组合查询预设价格
    + 支持实时价格数据的获取和缓存
    + 价格数据的分层查询：仓库级 → 全局默认 → 数据库默认
    + 价格查询失败时的降级处理机制

  - **价格数据缓存**：
    + 常用品类价格的本地缓存机制
    + 缓存数据的定期更新和同步
    + 网络异常时使用缓存数据的备用机制
    + 缓存数据的有效期管理和自动清理

- **价格应用逻辑**
  - **自动价格填充**：
    + 选择品类时自动填充对应的预设价格
    + 价格填充过程的动画效果和视觉反馈
    + 支持批量品类的价格自动填充
    + 价格填充完成后的确认提示

  - **价格计算集成**：
    + 计件数量变更时自动重新计算总价
    + 支持多品类计件的价格汇总计算
    + 实时显示单价、数量、小计和总计
    + 计算结果的精度控制和四舍五入处理

### 3.4 价格录入权限控制
- **智能权限判断**
  - **价格设置状态检测**：
    + 实时检测当前品类是否已有管理员预设价格
    + 区分不同司机类型的价格设置状态
    + 支持部分品类有价格、部分品类无价格的混合场景
    + 价格设置状态的可视化指示

  - **录入权限控制**：
    + 有预设价格：价格输入框自动锁定，显示预设价格，禁止修改
    + 无预设价格：价格输入框开放编辑，允许司机自主录入价格
    + 权限状态的实时切换和动态更新
    + 权限变更时的平滑过渡动画效果

- **管理员界面控制**
  - **输入框状态管理**：
    + 锁定状态：输入框灰色背景，显示锁定图标，价格不可编辑
    + 开放状态：输入框正常背景，显示编辑图标，价格可自由输入
    + 状态切换时的视觉反馈和动画效果
    + 鼠标悬停时显示状态说明提示

  - **权限提示信息**：
    + 锁定状态提示："此价格由管理员统一设置，不可修改"
    + 开放状态提示："管理员未设置价格，请手动输入"
    + 提示信息的友好展示和自动隐藏
    + 支持点击查看详细的价格设置说明

## 4. 品类管理仓库切换与价格设置功能
### 4.1 功能概述
- **功能gps**

### 5.1 功能概述

  - 为小程序所有界面提供统一的滑动返回导航体验
  - 通过右滑手势实现快速返回上一层界面的操作
  - 提供与微信小程序原生体验一致的导航交互

  - 详情页面和编辑页面
  - 列表页面和计件页面

### 5.2 滑动返回交互设计
- **手势识别机制**
  - **滑动方向**：从屏幕左边缘向右滑动
  - **触发区域**：屏幕左侧边缘20px宽度区域
  - **滑动距离**：滑动距离超过屏幕宽度的年龄%时触发返回
  - **滑动速度**：快速滑动时降低触发距离要求至屏幕宽度的20%
  - **手势冲突处理**：与页面内容的横向滑动手势进行优先级区分

- **视觉反馈设计**
  - **滑动指示**：滑动过程中显示半透明的上一层页面预览
  - **进度指示**：通过页面位移程度显示返回操作的进度
  - **阻尼效果**：滑动到一定程度后增加阻尼感，提供物理反馈
  - **回弹动画**：滑动距离不足时的回弹动画效果
  - **完成动画**：成功触发返回时的页面切换动画
